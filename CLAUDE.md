# CLAUDE.md

## Claude Code CLIへの指示

このプロジェクトでは、Claude Code CLIはLPの初回実装を担当します。

目的は、Image2で作成したLPデザイン画像を参考にしながら、Next.js / TypeScript / Tailwind CSSで、ポートフォリオ掲載できる品質のLPを実装することです。

参考画像をそのまま貼り付けるのではなく、HTML / CSS / Tailwindによって、Webページとして構造的に再現してください。

## このプロジェクトの前提

現在のプロジェクト場所は以下です。

`~/lp-portfolio`

Desktopではなく、Macのホーム直下に作成しています。

主なファイル・フォルダは以下です。

- AGENTS.md
- CLAUDE.md
- README.md
- project-setup.md
- references/memo.md
- notes/image2-prompt.md
- notes/claude-code-prompt.md
- notes/codex-prompt.md
- notes/portfolio-text.md

参考画像は `references` フォルダに保存します。

## Claude Codeの役割

Claude Codeは、主に以下を担当してください。

- LP全体の初回実装
- セクション構成の作成
- レイアウトの再現
- 配色の再現
- タイポグラフィの調整
- CTAボタンの実装
- PC / SP 両方のレスポンシブ対応
- 必要最低限のアニメーションやホバー演出
- Next.js / TypeScript / Tailwind CSSとして自然なコード作成

Claude Codeの役割は、最初から細部を完璧にすることではなく、LP全体をしっかり形にすることです。

細かい余白調整、コード整理、buildエラー修正、動きの微調整などは、後からCodexで行います。

## 絶対に守ること

### 1. 画像をそのまま貼らない

`references` フォルダ内の画像は、あくまで参考デザインです。

LP全体を画像として貼り付けることは禁止です。

以下の要素を、HTML / CSS / Tailwindで再現してください。

- ファーストビュー
- 見出し
- 本文
- CTAボタン
- セクション背景
- カード
- 装飾
- 余白
- 配色
- レイアウト
- レスポンシブ表示

画像は「完成イメージの参考」であり、実装はWebページとして作ってください。

### 2. 参考画像の雰囲気だけで終わらせない

参考画像を見て、なんとなく似た雰囲気にするだけでは不十分です。

以下を具体的に読み取って再現してください。

- どこがメインコピーか
- CTAはどこにあるか
- どの順番で情報が流れているか
- 余白は広めか、詰め気味か
- 色の主役は何か
- ボタンはどのくらい目立つか
- セクションごとの役割は何か
- スマホで見たときにどう並ぶべきか

LPは「きれいな画像」ではなく、「申し込みにつなげるページ」です。

## LPに入れたい基本構成

必要に応じて調整して構いませんが、基本的には以下の構成を想定してください。

### 1. ファーストビュー

目的は、最初の数秒で内容が伝わることです。

入れる要素の例：

- メインコピー
- サブコピー
- 体験レッスンの訴求
- LINE登録CTA
- やさしい雰囲気のビジュアル
- 初心者歓迎の安心感

ファーストビューでは、CTAが見つけやすいようにしてください。

### 2. 悩み・共感セクション

ターゲットが抱えている悩みを整理します。

例：

- 最近、体が重い
- 運動を始めたいけれど続くか不安
- 忙しくて自分の時間がない
- 肩こりや姿勢が気になる
- きつい運動は苦手
- でも少しずつ整えたい

不安をあおりすぎず、やさしく共感する表現にしてください。

### 3. ベネフィットセクション

ヨガを始めることで得られる変化を伝えます。

例：

- 呼吸が深くなる
- 姿勢を意識できる
- 体が軽く感じる
- 気持ちが落ち着く
- 自分の時間を作れる
- 無理なく続けられる

単なる機能説明ではなく、「生活がどう良くなるか」を伝えてください。

### 4. スタジオの特徴

スタジオの魅力をカード形式などで整理します。

例：

- 初心者歓迎
- 少人数制
- 女性が通いやすい
- 清潔感のある空間
- 駅近
- 体験レッスンあり
- LINEで簡単予約

視覚的に読みやすい構成にしてください。

### 5. 体験レッスン案内

体験レッスンの内容を分かりやすく伝えます。

例：

- 所要時間
- 持ち物
- 初心者でも参加可能
- 予約方法
- LINE登録特典

申し込み前の不安が減るようにしてください。

### 6. お客様の声・イメージ

架空LPとして自然な範囲で、お客様の声や参加者の感想を入れても構いません。

例：

- 初めてでも安心できた
- スタジオが落ち着いていた
- 仕事帰りに通いやすい
- 呼吸がしやすくなった

嘘っぽくならないよう、自然で短い文章にしてください。

### 7. 最終CTA

ページ下部に、もう一度LINE登録CTAを配置してください。

最後に背中を押すコピーを入れてください。

例：

- まずは体験レッスンから
- 無理なく、あなたのペースで始められます
- LINEから簡単に予約できます

## コピーの方向性

コピーは、強すぎる広告感を避けてください。

避けたい表現：

- 激やせ
- 絶対変わる
- 今すぐ痩せる
- 誰でも劇的変化
- 煽りすぎる表現

使いたい表現：

- 整える
- やさしく始める
- 自分のペースで
- 心と体をゆるめる
- 無理なく続ける
- 忙しい毎日に余白をつくる
- 初めてでも安心

## 実装方針

### Next.js

App Routerを前提に実装してください。

基本的には、まずLPとして見える状態を優先してください。

必要に応じて、以下を使用してください。

- `app/page.tsx`
- `app/globals.css`
- `components` フォルダ
- `public` フォルダ
- `references` フォルダ

ただし、構成を複雑にしすぎないでください。

### TypeScript

TypeScriptのエラーが出ないようにしてください。

Propsを使う場合は、型定義を分かりやすくしてください。

### Tailwind CSS

Tailwind CSSを中心にスタイリングしてください。

以下を意識してください。

- 余白を丁寧に取る
- スマホファーストで考える
- PCでは横並び、SPでは縦並びにする
- 角丸、影、グラデーションを上品に使う
- CTAボタンは視認性を高くする
- 背景装飾は控えめにする

## レスポンシブ対応

必ずスマホ表示を意識してください。

特に以下を確認してください。

- 横スクロールが出ない
- 文字が小さすぎない
- ボタンが押しやすい
- セクション間の余白が自然
- 画像や装飾が邪魔にならない
- ファーストビューで内容が伝わる
- CTAが見つけやすい

スマホで読みにくいLPは、完成度が低いと判断します。

## 動き・アニメーション

必要に応じて、軽い動きを入れてください。

例：

- ファーストビューのフェードイン
- CTAボタンのホバー
- カードのホバー
- セクションの軽い表示アニメーション
- 背景装飾の控えめな動き

ただし、動きは入れすぎないでください。

LPの目的は、内容を読んでもらい、LINE登録につなげることです。

アニメーションは、上品さと見やすさを高めるために使ってください。

## 画像について

参考画像は `references` フォルダに保存されています。

画像を使う場合は、以下に注意してください。

- LP全体を画像として貼らない
- 必要な場合のみ、部分的なビジュアルとして使う
- 画像が粗く見える使い方をしない
- 画像に頼りすぎず、CSSで再現する
- 装飾や背景はTailwindで作る

もし人物写真やイメージ画像が必要な場合は、仮のプレースホルダーでも構いません。

ただし、ポートフォリオ掲載前提なので、見た目が雑にならないようにしてください。

## 実装時に確認すること

実装後、以下を確認してください。

- LP全体が表示される
- ファーストビューが自然に見える
- CTAボタンが分かりやすい
- PC表示が崩れていない
- SP表示が崩れていない
- 文字が読みやすい
- 余白が詰まりすぎていない
- 色味が参考画像の方向性に合っている
- 画像をそのまま貼っただけになっていない
- buildエラーがない
- TypeScriptエラーがない

## Claude Codeへの期待値

Claude Codeには、以下の品質を期待します。

- 参考画像の雰囲気を読み取ったLP構成
- HTML / CSS / Tailwindでの構造的な再現
- ポートフォリオに載せる前提の見た目
- 初心者向けヨガLPとして自然なコピー
- CTAまでの流れが分かりやすい構成
- スマホでも読みやすいレスポンシブ
- 過剰ではない自然な動き
- 後からCodexで微調整しやすいコード

## やらないこと

以下は避けてください。

- 参考画像をそのまま貼る
- LP全体を1枚画像で表示する
- デザイン画像と関係ない雰囲気にする
- 色を増やしすぎる
- 強すぎるダイエット広告のようにする
- スマホ対応を後回しにする
- CTAが目立たない構成にする
- 文字が小さく読みにくい状態にする
- 意味のないアニメーションを大量に入れる
- コードを複雑にしすぎる
- buildエラーを残す
